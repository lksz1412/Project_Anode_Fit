## 2. 형식 판단 — bgbox 채택 (본문 (a)~(d) 소단락 아님)

**판단: `bgbox`[전이상태 이론…] 1개, 삽입 위치 = §5.1 (c) 문단 종료 직후(현행 42행 "…결과 박스를 닫는다." 문단 뒤, 44행 fig:barrier float 앞) + 본문 다리 1문장을 그 문단 말미에 추가.**

근거(정독 기반):
1. **§5.1 에는 이미 (a)(b)(c) 결과-박스 사슬이 산다** — eq:bv→eq:db→eq:logisticsolve 로 이어져 §5.2 의 (d) eq:xieq 박스를 닫는 한 사슬(rubric B1). 같은 소절에 제2의 본문 (a)~(d) 사슬을 삽입하면 단락 헤더가 이중이 되어 어느 (b)가 어느 사슬 소속인지 독해가 흔들린다. 박스 경계로 시각 분리하는 bgbox 가 유일하게 안전.
2. **rubric D3' [D8] 분기 조건 충족** — 이 유도는 독립 유도 포함·6문장 이상이므로 "길면 bgbox" 분기에 해당. 내용 성격도 배경(본문 결과식 불변, k₀·ΔS_a 의 기원만 보강)이라 keybox(핵심 결과)·derivbox(본문 유도) 아닌 bgbox 적격. §5 keybox 쿼터는 §5.3 이 이미 소진(D1').
3. **자족 블록 요건(D2')과 D-2 유보 대응** — DIRECTION 보고서 대안 배치(Part 0 §2.2 말미)로의 이동 가능성이 명시돼 있으므로, 어디로 옮겨도 성립하는 자족 블록이 요구됨. 본문 (a)~(d) 소단락은 이동 불가능한 형식이라 부적합.
4. **삽입점이 (a) 인용 직후가 아니라 (c) 문단 뒤인 까닭** — DIRECTION 문면은 "eyring1935 인용 직후"이나, (a) 단락 중간 삽입은 eq:bv 도달 전에 유도 사슬을 절단한다. Part 0 의 bgbox 전례(페르미온·보손, ch1_sec02a 171행)도 (d) 박스+verifybox 가 닫힌 **유도 휴지점**에 놓였다. §5.1 의 휴지점은 (c) 문단 끝이며, 본문이 박스 없이도 읽힌다는 D2' 원칙("정지점→logistic 흐름은 박스 없이 완결")에 이 위치가 부합. orphan 방지(A3)는 (c) 문단 말미 다리 1문장이 담당(페르미온·보손 전례의 "…별도의 배경 박스가 자족적으로 정리한다" 패턴 답습).
5. **bgbox 내부 구조** — 기존 관례(`\textbf{제목 ---}` 산문 시작)를 따르되, 유도 단계는 박스 안 `\textbf{(a) 출발 ---}`~`\textbf{(d) 박스 ---}` 미니 헤더로 표시(rubric B1 규율을 박스 내부에도 동일 적용; Part 0 의 (a′)(b′) 프라임 전례처럼 사슬 구분은 표기로 해결되고, 박스 경계가 본문 사슬과의 혼동을 차단). 검산은 **verifybox 분리가 아니라 박스 내부 말미 `\textbf{검산·연결 ---}` 단락** — verifybox 를 박스 밖에 두면 본문이 박스 없이 읽히지 않게 되어(검산이 박스 내용을 참조) 자족성이 깨진다. 페르미온·보손 bgbox 도 극한·가드를 내부 산문으로 처리한 동일 전례.

## 3. 유도 자체 재검산 기록 (전 식)
