- **v4→v5: 깊이 보존, 표현 밀도 전환.** 번호식 0 손실 + 유도 명시성 상승(승격식 5 — eq:affinity 이중계상 논거·chisum ∀𝒜 양화 등 산문에 숨어 있던 논증의 수식화)으로 **식 사슬 층위의 깊이는 v3/v4와 동일하거나 국소적으로 더 촘촘**. 반면 "왜 이 항이 여기 오는가"의 물리적 서사층(분포의 기원, plateau의 앙상블 해석 등)은 얇아짐 — 검수가 load-bearing 단서 일부를 복원해야 했음(V5 ledger R4 ② "공존 plateau는 다입자 앙상블이 받침" 1문장 — v4 keybox에서 증발했던 것). §1.18 배제로 히스테리시스 미시층은 v3 수준으로 회귀.

- **v3(Fable) = 교과서 문체의 실현체**: 완결 문장 서사, 절 도입 다리(선행 결과식 번호 인계, X.2)·절 마무리 keybox 요지, worked-example 내러티브, 해설 후치 원칙(v3 plan 변환 원칙 2 — "물리 해석 prose 는 삭제하지 않고 식 뒤로"). 사용자 6차 피드백("교과서처럼")의 직접 산물.
- **v4(Opus) = 문체 계승**: §1.18도 동일 문체(도입 동기 문단·완결 서사·keybox)로 작성 — 문체 전환 없음.
- **v5(Opus) = 별도 장르(수식-구동)**: 서술 문단 금지 + 커넥터 화이트리스트("둘째 등호는 대수적 재배열이다", "정지점 dξ/dt=0 에서 … 이고, 그 비는 식 (1.22) 이므로" — v5 L546·552 실물). 같은 내용의 두 표현을 대조하면: v3 §1.11 도입 "재료가 모두 준비되었다. 평형 기준선과 세 가지의 결과 … 를 관측식" ↔ v5 L1160 "평형 기준선~(1.51) 과 세 가지의 결과 — 꼬리~(1.63)·유효 장벽~(1.64)·분포 일반형~(1.73) — 를 관측식~(1.47) 위에서 합치면 … 닫힌 함수가 된다" — 서사는 죽고 식번호 그물은 살았다. 전보체 재발 0·커넥터 misstatement 0은 검수로 확인(V5 ledger R2).
- 유의: v3 문체와 v5 문체는 **둘 다 사용자 명시 주문**(각각 6차·v5 요청)이라 우열이 아니라 용도 분화다. 문제는 문체가 아니라 §3.5의 "park 구조"(한 장르가 계보의 유일 baseline이 되는 것)에 있다.

## 7. v12에 물려줄 교훈

1. **확장의 모범형 = v3→v4**: 기존 본문 바이트 불가침 + 국소화된 diff(기호표 행·훅·별도 절) + 하위호환 게이트 병행. v12에서 이론 수정을 하더라도 "무엇이 원본이고 무엇이 수정인지"가 diff 한 방에 나오는 구조를 유지할 것.
2. **무계획 배치 Write는 실결함을 낳는다(실증)**: V4.1의 σ 기호 충돌·B/κ 산수 오류·기호표 누락은 전부 통째 작성+검수 1회의 산물이었고, 단위 루프+N-round가 이를 적발했다(커밋 67ab178의 자기 기록). v12 작성(N=10 체리픽)에서도 단위 구성 루프·수렴 검수는 타협 불가.
3. **수렴 후 편집 = 게이트 재실행 의무(실증)**: V5.R8 3줄 수정이 overfull 2를 유입시키고도 커밋 메시지는 "overfull 0"을 주장 — 게이트 주장은 실행 증거 없이는 신뢰 불가(gate 설계 원칙과 정합). v12의 "연속 2R 0결함" 딱지는 마지막 편집 *이후* 시점 기준으로만 유효.
4. **장르 전환은 자산을 지운다 — 병합 설계 필수**: v5는 keybox 13개·stagebox·동기 산문·athermal 훅을 스펙에 따라 지웠고, 이후 계보가 v5 위에서 진행됐다. v12는 "수식-주도 + 교과서 문체" 두 축을 모두 요구하므로, **v5의 식 사슬(97식 verbatim 검증 자산·의존 그래프) + v3/v4의 다리·요지·동기 층을 명시적 병합 대상으로 계획에 적시**할 것 — 어느 한 판본의 단독 계승은 반드시 한 축을 잃는다.
5. **park된 물리의 재개방 결정 명시**: v4 §1.18(적층 준안정)과 §1.16 athermal 절편 바닥 논점은 "배제"가 아니라 "보류"였다(V5RR RESULT §11 "차기 모델 재개방 시 별도"). v12 Phase 4.1 Decision Gate에서 재개방/영구 배제/부록화 중 하나로 **명시 결정**하고 기록할 것(암묵 잊힘 방지). 히스테리시스 절편의 온도 검정 논의는 athermal 바닥 유무에 따라 반증 설계가 달라지므로 물리적으로도 미결이 아니라 결정 사항이다.
6. **v3도 무결 아님 — 산문이 메우던 다리 3곳**: (1.95)·(1.75)·(1.83)은 Fable 원천 설계의 암묵 의존이었고 장르 전환이 이를 노출했다. v12 재작성 시 이 3곳은 번호식 또는 명시 다리로 완결할 것(V5RR의 cross-ref 보강은 완화이지 종결이 아님).
7. **버전 번호 충돌 관리**: 구트랙(canonical, 5-28)과 본 계보(tex, 6-12~)의 v3/v4/v5는 동명이물 — 모든 후속 감사·계획 문서에서 버전 언급 시 파일 경로 병기(본 노트 §1.3).
8. **attribution 규율**: v4의 Fable_v4→Opus_v4 개명 사례처럼 작성 주체가 바뀌는 지점에서 파일명·헤더의 작성자 표기를 즉시 정합시킬 것 — v12는 다중 모델 체리픽(3S+3O+3C+1F)이므로 드래프트별 작성자 기록이 더 중요해진다.
