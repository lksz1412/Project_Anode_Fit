## 0. 요지 (한 문단)
계보는 "**축소→재확장 왕복**"이다: Fable_v3(적극적 무결 기록 보유, 200KB)에서 v4(순수 추가)·v5(수식-구동 장르 전환, 식 보존)·v6(흐름도 재조립, 손실 0)까지는 자산이 보존됐으나, **v7 재출발이 코드 플로우차트를 척추로 의도적 절삭**(894줄, "곡선에 필요한 식만")했고, 사용자가 사후 "중간 도출을 날렸다"고 판정해 v8(유도 복원)→v9(LCO)→v10(broadening 복원)→v1.0.10(코드-문건 동기화)으로 **선별 재확장**됐다. 이 왕복에서 물리 결과식·부호는 잘 지켜졌으나(각 세대 검증 기록 실재), ★**사용자 방향 지시 중 미계승분이 남았고**(Eyring 척추·역방향 식별 사슬·적층 준안정 등), 프로세스 측면에선 **"설계 doc 순응 검수의 한계"와 "실행 기반 검증의 결정력"**이 반복 실증됐다. v1.0.11은 v1.0.10과 byte 동일(P1.1 supplement 미편입 중단).

## 3. 관통 패턴 — 프로세스 감사 (확정)
1. **★설계 doc 순응 검수의 한계**: 리뷰어들이 "설계서와의 내부 정합"만 채점해, 설계서 자체의 물리 오류(w_eff narrowing)가 adversarial 2라운드를 통과(A5). R1 오판도 동류(SPEC 미대조 채점, A4). → **설계서 자체를 검증 대상으로 + 기록(SPEC) 대조 의무화**.
2. **★실행 기반 검증의 결정력**: w_eff 반증·bell/폭 실측·R1 철회·factor-2 적발 전부 **코드 실행 실측**이 해냈다. 기호·정합 검수만으로는 "일관되지만 틀린 답"에 수렴 가능. → 검수 렌즈에 실행층 상설.
3. **통째 배치 작성 = 사고 재현**(V4.1 σ 충돌 29개소·v10 기작③ 단발 저작 자기모순) ↔ 단위 루프·경쟁 다양성은 결함을 표면화(v7 부호 4/9·v9 ΔS_e 5건·spine 오류 적발).
4. **수렴 선언 후 편집은 게이트 재실행 필수**(v5 R8 overfull 회귀 + 커밋 메시지 허위 — 신뢰 회계 문제).
5. **다중 모델 운용 마찰**: 오명명(Fable_v4)·Codex 관찰 부실(one-shot 방치→유실 2회)·attribution 흐림 — 스킬 §3 단계구동+관찰 및 즉시 정합 필요.
6. **문서-코드 이중 진실**: 코드는 v11_final서 사실상 동결(폭 물리 미변경)인데 문건은 v8~v10서 크게 진화 — 정합은 v1.0.10에서야 맞춤. 이 시차가 R1 오판의 토양.
