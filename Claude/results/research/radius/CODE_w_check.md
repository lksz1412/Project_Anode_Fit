> 그림: `fig_bell_vs_spike.png`(코드 실제 출력). 코드 원형 불가침("1바이트도 변조 X") — 본 문건은 *관찰·보고*만, 코드 수정 X.

## 측정값 (단일 LiC₆ 전이, equilibrium(), T=298.15K)
| 경우 | peak 높이 | FWHM | 면적(Q=0.5여야) | 판정 |
|---|---|---|---|---|
| 기본 w=12 mV | 10.42 | **42.3 mV** | 0.499 | ✓ 정상 종(area 보존) |
| **코드 use_w_eff** (w 없음·Ω=13000) | 97.31 | **90.6 mV** | **9.294** | ✗ 더 넓고 18.6× 부푼 깨진 종 |
| 의도된 w_eff (n=0.05 일관 적용) | 97.31 | 4.5 mV | 0.500 | 진짜 뾰족이(코드는 안 함) |

## 결론
1. **코드는 뾰족이가 아니라 종모양을 그린다.** 기본 출력(full dQ/dV: 4 transition + kinetic tail)은 구조적 종 곡선. 단일 전이 = FWHM 42 mV 정상 종, 면적=Q 보존. → "뾰족이만 나온다/비정상 그래프"는 기본 상태에서 일어나지 않음.
2. **use_w_eff는 뾰족이를 만들지 않는다 — 오히려 깨져 있다.** ξ_eq의 logistic 폭은 `func_ksi_eq` 내부에서 `func_w(T,n)`로 계산되는데, w_eff 경로(`'w'`·`'n'` 없음)에서 `_n_factor`가 **n=1**을 돌려 폭=`RT/F`=25.7 mV(넓음). 그런데 진폭은 `÷w_eff(1.28 mV)`. **둘이 어긋나** 폭은 그대로(오히려 25.7mV로 기본보다 넓음)이고 진폭만 ~18배 부풀어 **면적 보존(∫dξ=1)이 깨진다**(area 9.3 vs Q 0.5). 즉 w_eff는 (a) 물리적으로 거꾸로(Ω↑→델타로 narrowing)일 뿐 아니라 (b) 코드 구현이 ξ_eq 폭과 분모 w 불일치로 **버그**.
3. ★코드 정합 조건: `_width`와 `func_ksi_eq`의 폭이 일치해야 면적이 보존됨. 기본 경로는 `_n_factor`가 'w'에서 n=w·F/RT를 역산해 `func_w(T,n)=w`=분모 w → **일치(정상)**. w_eff 경로만 불일치.

## 함의
- **w는 강한 1차 전이(이 staging 4개 전부 Ω>2RT)에서 자유 피팅 파라미터**가 맞다(v4/v5 "현상학적 피팅 폭"). 코드 기본도 명시 w(12~20mV)+use_w_eff=False로 이미 그렇게 씀.
- **use_w_eff는 비활성/제거 권장**: 켜도 staging엔 'w'가 있어 적용 안 됨(이중 가드). 'w' 없는 전이에 켜면 깨진 넓은 종. 의미 없음.
- **v8 정정 필요**: w_eff를 "상호작용이 종을 좁힘"으로 유도해 두었으나, 코드는 그렇게 구현하지 않고(불일치) 물리적으로도 거꾸로. v8은 w를 두-상=자유 피팅 폭 / 단상=nRT/F 평형 예측으로 구분하고 w_eff 절을 (제거 또는 "Ω<2RT 단상 한정 + ξ_eq 폭과 동시 적용해야 면적 보존" 경고와 함께) 바로잡아야.
