## A. 확정 결함 (severity / 위치 / 무엇이 틀림 / 옳은 형태)

### CRIT-1 — 동역학 꼬리의 "전위 의존 L↓" 가 v11 코드에 없음 (spec/code divergence)
- **위치**: §"전위 의존: V↑⇒L↓" eq (1.21)·(1.22) (tex L556–564), 그리고 그에 의존하는 §S3 eq (1.28) (L738–744).
- **무엇이 틀림**: 문건은 `A_j = sF(V_n−U_j)` (전위에 비례하는 affinity)를 전제로 `∂lnL_q/∂V_n ≈ −χ_d F/RT ≤ 0`(전위가 깊어질수록 꼬리 짧아짐)을 유도한다. 그러나 **v11 코드의 A 는 전위 무의존 상수**: `A = min(z_cut·n·RT, A_cap·RT)` (L335), 한 전이당 **스칼라 1회** 산출(L463 `lag_len_V` = scalar, V_work 격자 위에서 재평가 안 됨). 코드 주석 L329 가 명시: 원본 `s·F·(OCV−U)` 꼴은 **상수 컷 affinity 로 교체됨**. 즉 eq (1.21)/(1.22) 는 v11 이 **버린** 옛 형태를 설명한다. v11 은 한 전이 안에서 L_q·L_V 가 V_n 에 따라 변하지 않는다.
- **연쇄 피해(G-usable)**: §S3 (eq 1.28) `χ = −(RT/F)·∂lnL_q/∂V_n` 식별 프로토콜은 코드가 평평하게(상수 L_V) 내놓는 양의 기울기로 χ 를 읽으라 지시 → 이 문건만 보고 v11 을 재현·피팅하려는 독자가 코드와 모순된 절차를 따른다. brief §4 "이 문건만으로 v11 재현"·brief §8(6) 위반.
- **옳은 형태**: (a) A 를 상수 컷 affinity 로 서술하고 L_q 를 전이당 상수로 명시하거나, (b) "전위 의존"·S3 식별이 v11 의 구현이 아닌 *물리적 동기/외삽 모델*임을 명확히 분리. 현재는 코드 식(eq:lnLq L542)과 같은 절에서 마치 코드 거동인 양 제시되어 오인 유발.

### HIGH-1 — 보편식 L_q(eq 1.20)이 코드의 ΔS_a(−TΔS_a) 항을 누락
- **위치**: 박스식 eq (1.20) (tex L533–537).
- **무엇이 틀림**: 코드 L96 `dG_a = dH_a − T·dS_a`, `ln_Lq = … + dG_a/RT − x·A/RT` → 지수에 `(ΔH_a^eff − T·ΔS_a − χ_d A)/RT`. 문건 박스식은 `e^{(ΔH_a^eff − χ_d A)/RT}` 로 **−TΔS_a(=절편 e^{−ΔS_a/R}) 누락**. 정작 같은 문건 로그형 eq (1.21) 의 `b_j = −ΔS_a/R + …` 에는 들어 있어 **자체 모순**(닫힌형 ≠ 로그형). GRAPHITE_STAGING_LIT 는 dS_a=0 이라 수치만 우연히 일치.
- **옳은 형태**: 박스식 지수를 `(ΔH_a^eff − T·ΔS_a − χ_d A)/RT` 로(또는 명시적으로 `·e^{ΔS_a/R}` 선인자) 정정해 코드 L96·로그형 (1.21)과 일치시킬 것.

### MED-1 — N6 평형 peak 위치를 U_j^d 로 단정, 코드 equilibrium() 는 U_j 사용
- **위치**: eq (1.13) `V_peak = U_j^d(T)` (tex L462) + L468 "코드 equilibrium()(L354)는 식 (1.12)의 직접 구현".
- **무엇이 틀림**: 코드 `equilibrium()` (L354–371)은 `func_ksi_eq(T, V, U_j, n_j)` 를 **U_j(분기 없음)·기본 s=+1** 로 호출 → 기준선 peak 는 **U_j 에 선다(U_j^d 아님)**. dqdv 본경로(L459)는 center=U_j^d 라 거기서는 (1.13)이 맞지만, "equilibrium()=(1.12)의 직접 구현"이라는 동일시는 U_j vs U_j^d 에서 어긋난다(γ≠0 전이일 때).
- **옳은 형태**: 평형 기준선은 분기 미반영(U_j)임을 명시하고, U_j^d peak 는 본경로 한정으로 구분.

### MED-2 — spinodal 과주행 방향이 v5 레퍼런스와 반대
- **위치**: eq (1.5) 직전 서술 (tex L320–322): "방전(ξ↑)은 ξ_s^+ 까지, 충전은 ξ_s^- 까지".
- **무엇이 틀림**: v5 정본 L1337 은 정반대로 "방전(탈리튬화)은 극대 ξ_s^- 까지, 충전은 극소 ξ_s^+ 까지"라 적는다(NOTE.md R8 에서 저자가 v5 와 어긋나게 "정정"). **단, 코드 영향 0**: func_U_branch 는 ΔU_hys 크기와 ½σ_d 부호만 쓰므로 ξ_s^± 귀속은 순수 해설. 그러나 같은 척추를 공유하는 9편/원전 v5 와 상충하므로 일관성 결함으로 기록.
- **옳은 형태**: ξ 규약(탈리튬화 진행률)을 명시한 뒤 v5 와 동일 귀속으로 통일하거나, 규약 차이를 각주로 해소.
