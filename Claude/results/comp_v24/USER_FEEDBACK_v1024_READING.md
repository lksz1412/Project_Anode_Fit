## F-02 — §1.2.1 대문자 P 이중 사용(압력 vs 확률)
- **사용자 지적**: 식 1.6 은 대문자 P 를 압력으로, 식 1.7 은 바로 확률로 씀. 아래첨자가 있어도 같은 대문자 P 라 오해 소지 큼. 노테이션 정리 필요. **방향 제안**: 열역학·물리 상태함수의 통상 표기(압력 $P$ 등)는 그대로 두고, 확률처럼 저자마다 다른 기호만 조정. 확률은 **소문자 $p$** 가 분간에 도움.
- **위치**: `_sections/ch1_sec02a_part0.tex` · §1.2.1 "앙상블과 Boltzmann·Gibbs 인자"
  - 식 1.6 = `\label{eq:sm-fund}` (line 30–36): $\dd E=T\dd S-P\dd V+\mu\dd N$, $\tfrac{P}{T}\dd V$ → **P = 압력**
  - 식 1.7 = `\label{eq:sm-resv}` (line 46–49): $P_i\propto W_R(\cdots)$ → **$P_i$ = 확률**
- **현재 상태(원문 확인)**: 확률 P 는 이 절 여러 곳 — $P_i$ (line 47·60·77, 대정준 박스 `eq:sm-gc` 포함). 압력 P 는 근본 미분식(line 30·32). 참고: 문건은 유사 충돌(Helmholtz $F$ vs Faraday $F$)을 line 66 에서 "기호만 같고 문맥으로 갈린다"로 명시 처리한 전례가 있음 — 사용자는 확률의 경우 문맥 의존보다 소문자 p 로 분리하자는 취지.
- **분류**: 노테이션 충돌(압력 P vs 확률 P)
- **집행 시 고려(중립·미결정)**: 확률 $P_i\to p_i$ 치환은 이 절뿐 아니라 대정준 박스 `eq:sm-gc`, 하류의 확률 참조 전부에 파급 → 전수 치환 + 소문자 p 가 다른 양과 겹치지 않는지 재검 필요. 압력 P 유지는 사용자 방향과 일치. **검수 7항목 #1(노테이션 일관성)**과 직접 연동.
- **상태**: 접수(미착수)

## F-03 — 식 1.21 소문자 f(Helmholtz)·s(엔트로피) 의 직관성
- **사용자 지적**: 식 1.21 은 Helmholtz 자유에너지를 소문자 $f$ 로 써서 직관적이지 않음. 통상 표기가 맞는지 의문. 이로 추산하는 내부 엔트로피 $s$ 도 소문자라 직관적이지 않음.
- **위치**: `_sections/ch1_sec02a_part0.tex:320–325` · 식 1.21 = `\label{eq:sm-sint}`
  - $f_\mathrm{int}=-k_BT\ln q$ (line 320, 소문자 f), $s_\mathrm{int}=-\partial f_\mathrm{int}/\partial T$ (line 322, 소문자 s)
- **현재 상태(원문 확인)**: 이 $f_\mathrm{int}$·$s_\mathrm{int}$ 는 명시적으로 **자리당(per-site) 내부 자유도** 양(line 319 "내부 자유도의 자유에너지 몫", 자리 하나 기준). 문건은 이미 **총량(extensive)에 대문자**를 씀 — 총 Helmholtz $F=-k_BT\ln Z$ (line 64), 총 엔트로피 $S=k_B\ln W$ (line 27). 즉 현재는 **대문자=총량 / 소문자=자리당(intensive)** 규약이 서 있음.
- **분류**: 노테이션 직관성(소문자 자유에너지·엔트로피)
- **집행 시 고려(중립·미결정)**: 단순 $f\to F$·$s\to S$ 승격은 기존 총량 $F$·$S$ 와 정면 충돌. 사용자 방향("직관적 대문자")과 기존 규약(자리당 소문자) 사이 해소책 예: (i) 소문자 유지하되 첨자·정의로 "자리당" 강조, (ii) $\tilde F,\tilde S$ 또는 $f_\mathrm{site}$ 등 명시적 per-site 기호, (iii) 다른 관례 도입 — 어느 쪽이든 **총량 $F,S$ 와의 구분 보존이 조건**. 최종 선택은 사용자. (F-02 확률 $p$ 방향과 함께 "노테이션 일괄 정리 패스"로 묶어 집행하면 효율적일 수 있음 — 후보.)
- **상태**: 접수(미착수)

- **집행 시 고려(중립·미결정)**:
  - **밀도 완화 레버(각각 "약간씩")**: 줄간격 1.12→~1.15–1.2 · 문단간격 0.45em→~0.6em · **여백 22mm→~26–28mm**(행 길이 단축 = 촘촘함 완화의 최대 레버) · `microtype` 도입(XeLaTeX는 **protrusion 작동**, expansion/tracking은 제한 — 본문 tracking(글자 간격 벌리기)은 과하므로 protrusion 위주). 실제 수치는 집행 시 사용자 승인.
  - **줄바꿈/띄어쓰기 두 의미 분리**: (a) 조판상 overfull/줄나눔 — `emergencystretch`·`microtype`로 개선, (b) **한글 맞춤법 띄어쓰기 정오** — 별도 교정 패스(문체 F-04 와 함께 묶기 가능).
  - **인라인→디스플레이 승격**: 문단 내 복잡 인라인 수식을 `\[...\]`/`equation`으로 승격. **복잡도 기준 필요**(분수 중첩·합/적분 기호·다항·조건부 등은 승격, 단항 기호·짧은 관계식은 인라인 유지). 문서 전역 per-instance 판단 → 대규모.
  - **파급(중요)**: 전역 간격/여백 변경은 **문서 전체 리플로우** → 페이지 수 변동(현 ch1 91p·ch2 28p·ch3 20p 바뀜) → `HANDOVER`·`INDEX`·`MERGE_READINESS`의 페이지 수·페이지 참조 갱신 필요. **빌드 GREEN 유지 필수**(0 err·0 undefined).

## F-07 — E.3 서지 enumerate 라벨이 왼쪽 페이지 밖으로 잘림 (레이아웃 버그)
- **사용자 지적**: E.3(서지 관련 내용)이 **페이지 밖으로 일부 나가 있음**.
- **위치**: `_sections/ch1_appE_selfconsistent.tex` §E.3(`sec:sc-p35`) · 컴파일 **PDF 85쪽** · `enumerate` 라벨 line 111·113·117·120·124.
- **근본 원인(빌드+렌더 직접 확인)**: 5개 항목이 `\item[\textbf{①\ 서지.}]`처럼 **긴 볼드 문구를 커스텀 라벨**로 씀. 라벨이 `\labelwidth`보다 넓어 **왼쪽으로 흘러나가고**, 리스트가 좌측 여백에 붙어 있어 넓은 라벨(②③④⑤)의 **원문자·앞부분이 페이지 좌측 끝에서 잘림**. 85쪽 렌더에서 확인: ①만 온전, **②→"논문…"(② 잘림)·③→"방법…"·④→"…핑(graphite lag)"("④ 변수 매" 잘림)·⑤→"…(그대로 이식 금지)."**.
- **중요(게이트 공백)**: 이 왼쪽 흘러넘침은 **`Overfull \hbox` 경고를 내지 않음**. 그래서 "GREEN 빌드(0 err·0 overfull-warning)" 기준이 **놓쳤음**. → 집행 시 **렌더 기반 시각 점검**을 게이트에 추가 권고.
- **범위(전수 확인)**: `\item[\textbf{...}]` 넓은 라벨 구문은 **문서 전체에서 정확히 5곳, 모두 E.3**(다른 절엔 없음). 버그는 E.3 로 국한 — 수정 범위 명확.
- **분류**: 레이아웃/조판 버그(국소)
- **집행 시 고려(중립·미결정)**: 수정 방향 예 — (i) 커스텀 라벨을 없애고 **본문 앞 볼드 리드인**으로(`\item \textbf{① 서지.} 적용 논문 …`) 일반 enumerate/itemize, (ii) `description` 계열로 **행걸이(hanging)** 처리, (iii) 해당 리스트만 `\leftmargin`·`\labelwidth`·`\labelsep` 확대. 물리 내용·`\cite`·`\label` 불변. 어느 방식이든 85쪽 재렌더로 확인. (F-06 조판 패스와 함께 처리 가능.)
- **관련 데이터(참고 — F-06 넘침 스윕용)**: ch1 빌드 로그의 **텍스트 overfull \hbox 13건**(줄번호 有). 실질 우측 넘침: `ch1_appE:196–206`(E.6 코드지도, 7–46pt)·`ch2_sec07_revheat:75–81`(30pt)·`ch2_appA_traps:63`(24pt)·`ch1_sec01:59`(6.6pt)·`ch2_sec00_intro:6–15`(6pt)·`ch2_sec02_config:156–166`(2.5pt). 별개로 `ch1_appB_codemap:62–102` **568pt ×3**(코드맵 longtable 내부 — 넓은 `\code{}` 행, 표 처리 별도). ※ 일부는 **Ch2** — 해당 챕터 정독 시 사용자 확인 후 함께 처리(지금 front-run 안 함).
