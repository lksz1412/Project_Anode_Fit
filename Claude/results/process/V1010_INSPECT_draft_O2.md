## 1. 문제 목록 (recall 우선 — 의심 포함, 오적발 가능성은 §4 자기표시)

### [CRIT-1] 흑연 4-staging 전이가 단일 bell 로 병합 — 실측 피크 1개 (렌즈 1·3·4)
- **위치**: `equilibrium()`(L372-389)·`dqdv()`(L453-499) + `GRAPHITE_STAGING_LIT`(L680-709) + 커밋 그림 `anode_fit_v1_0_10_dqdv.png` 패널(1)·`P5_graph_suite.png` V1·V5.
- **실측**: 4-staging equilibrium(U=0.210/0.140/0.120/0.085) 을 4000점 격자로 그려 국소 최대를 세니 **피크 1개**(V≈0.100). 목표 4개 staging 전이가 **완전히 병합된 단일 skewed bell**. 커밋 그림 3종 모두 동일(방전·온도의존·율의존 전부 단봉).
- **무엇이 틀렸나**: 실측 흑연 dQ/dV 는 저율서 **3~4개 뚜렷·날카로운 staging 피크**(4→3, 3→2L, 2L→2, 2→1)를 보인다. 코드 개형은 이 특징을 재현 못 하고 하나의 완만한 종으로 뭉갠다.
- **왜 문제**: 근본 원인은 **모든 전이가 동일 폭 nRT/F≈25.7 mV 를 쓰는데 U_j 간격(0.210↔0.085=125mV, 인접은 20~30mV)이 폭과 같은 규모**여서 인접 종이 서로 삼켜진다. 이것이 사용자가 P1-P5 가 놓쳤다고 지목한 "뭉갠 단일 bell·병합" 바로 그 현상. → CRIT-2 와 한 뿌리.

### [CRIT-2] 폭 w 가 Ω(상분리)·broadening 과 무관한 열역학 단일자리 값에 고정 (렌즈 4 = 극한 과도 일반화; 렌즈 1·3)
- **위치**: `func_w`(L74-75)·`_width`(L303-306)·`_n_factor`(L294-300) + `GRAPHITE_STAGING_LIT` 각 전이 `'n':1.0`.
- **실측**: 같은 LiC6 전이에서 Omega 를 0·6000·13000·30000 J/mol 로 바꿔도 **FWHM=90.07 mV 로 완전히 불변**(4회 동일). 즉 폭이 Ω 에 전혀 반응 안 함 → 폭 = 순수 `nRT/F`.
  - 단일 종 FWHM 이론 = logistic 도함수 반높이 z=±ln(0.85355/0.14645)=±1.7627 → FWHM=3.526·(RT/F)=**90.57 mV**, 코드 측정 90.57 mV **정확 일치**(면적 = Q 정확 보존, 매우 넓은 격자서 0.500000). → 코드 종 = 정확한 logistic 도함수 종.
- **무엇이 틀렸나**: 흑연 staging 은 **1차(두-상, Ω>2RT) 전이**다. 평형 단일입자 dQ/dV 는 Maxwell 작도상 plateau 안에서 near-delta(spike)여야 하고, 실측 유한 폭(수십 mV)은 **평형이 정하는 값이 아니라** broadening(율속 꼬리 + 내재 RT/F + 앙상블 apparent-U 이질성)에서 온다. 그런데 코드는 폭을 **열역학 단일자리 값 nRT/F 로 고정**해 넣는다 — 이는 사용자 렌즈 4 의 정확한 사례("단일자리 Nernstian w=nRT/F 를 1차 두-상 전이에 보편 적용, delta 여야 할 걸 bell 로").
- **왜 문제 (★code↔doc 물리 gap)**: Ch2 v1.0.10 문건은 이 지점을 **정확히 알고** 있다 — §2.6 파생C(L539-568)·종합식 warnbox 가 "두-상 전이의 w_j 는 평형 예측 nRT/F 가 아니라 **다온도 dQ/dV 로 피팅하는 현상학적 자유 폭**"이라 못 박는다. **문건은 옳게 서술하나 코드는 그 폭을 실제로 피팅하지 않고 nRT/F 하드 디폴트로 출고한다**. 문건(피팅 폭)과 코드(고정 열역학 폭)가 어긋난다. FITTING_GUIDE tier-1 은 n 을 [>0,~4] 자유폭으로 열어두지만 **출고 staging 값은 전부 n=1** → out-of-box 개형이 CRIT-1 의 병합 bell.
- **radius 참조(회의적, refute 통과)**: RADIUS_VERDICT L18 도 독립적으로 "기본 use_w_eff=False, 기본폭 nRT/F≈25.7mV" 확인, L27-32 는 마이크론 흑연서 반경→U_j 결합이 실측 폭의 ~1/1000 이라 폭이 반경분포서 오지 않음을 명시. 이는 "폭의 물리 기원이 모델 밖"이라는 내 실측과 삼각검증됨(단 radius 의 반경가설 자체는 채택 안 함).

### [HIGH-3] 출고 staging 의 per-transition 'w' 값이 silent 무시 (렌즈: silent path·G-usable)
- **위치**: `_n_factor`(L294-300) 우선순위 + `GRAPHITE_STAGING_LIT`(L681-708).
- **실측**: 4 전이 모두 `'n':1.0` **과** `'w':0.020/0.016/0.014/0.012` 를 동시 보유. `_n_factor` 는 'n' 우선이라 'w'(12~20mV)가 **전부 무시**되고 4 전이 모두 동일 25.69mV. 코드 주석(L9-10)도 "'w' 폴백은 'n' 존재 시 inert" 라 인지하나, **데이터셋이 서로 다른 w 로 전이별 폭 차등을 의도**한 흔적(0.020→0.012 감소)이 죽어 있다.
- **왜 문제**: 저자가 넣은 전이별 폭 차등(작은 stage 일수록 좁게)이 침묵 상실 → 개형 표현력이 의도보다 낮다. 사용자가 데이터 tweak(w 조정)해도 'n' 지우기 전엔 무반응 = G-usable 함정. (등급 HIGH: 물리 오류라기보단 사용성·의도 손실이나 CRIT-1 병합을 악화.)

### [MED-4] entropy_coefficient 가 doc 의 전하보존 음함수(eq:implicit)를 풀지 않고 V-축 직접 가중 (렌즈 2·G-follow)

- **무엇**: 문건 "use-this" 절차는 "①전하보존 Σ_j Q_j ξ_j(U_oc,T)=Qx 를 풀어 U_oc(x,T) 획득 → ②그 U_oc 를 ξ_j 에 되먹임" 을 명시한다. 코드는 이 음함수 풀이를 **하지 않고**, 모든 전이의 g_j=ξ_j(1-ξ_j)/w 를 **같은 입력 V 에서** 평가해 가중한다.

## 2. ★가장 약한 1곳 (필수 지목)

**CRIT-2 (폭 w 를 열역학 nRT/F 로 고정) 가 최중대·최약점.** 이것이 CRIT-1(4-staging 병합 단일 bell)의 직접 원인이며, 사용자가 P1-P5 가 놓쳤다고 지목한 "물리 오류"의 핵심이다. 물리 실체: 흑연 staging = 1차 두-상 전이 → 평형은 near-delta, 실측 유한 폭은 **broadening 이 결정**. 코드는 폭을 단일자리 Nernstian 열역학 값에 고정해(렌즈4 극한 과도 일반화) 병합 bell 을 낳는다. **결정적 아이러니**: Ch2 문건은 이 물리를 정확히 서술(w_j=현상학적 자유 피팅 폭, v4 의 틀린 w_eff 축소식까지 제거)하는데, **코드는 그 서술을 이행 안 하고** nRT/F 하드코딩 + 출고 staging n=1 로 병합 개형을 출력한다. v1.0.11 은 (a) staging 폭을 broadening 반영 자유 파라미터로 실제 개방(출고값을 실측 유한폭 ~10-15mV 수준으로 재설정하거나 n>1 로), (b) 4 피크가 분해되도록 폭 vs U_j 간격 정합, (c) 문건-코드 폭 지위 일치 — 이 세 가지를 최우선으로 다뤄야 한다.

## 3. cross-version 누락 판정 (담당 Ch2 v3·v4 → v1.0.10)

### v4 파생C `w_eff(Ω)=w(1−Ω/2RT)` (v4 §2.6 eq:weff L531-580) → v1.0.10 에서 **삭제** = **의도적 컷·정당(결함 아님)**
- **무엇이 컷됐나**: v4 는 상호작용 Ω 가 봉우리를 **좁힌다**(w_eff<w, Ω→2RT 서 w_eff→0=delta)고 전개. v1.0.10 헤더(L8-9)·§2.6 warnbox(L561-568)가 이를 **완전 제거**하며 사유 명시: "two-phase 실측은 종이지 델타가 아니라 narrowing 그림이 틀림 — Ω 가 봉우리를 좁혀 델타로 만든다는 서술은 실측 방향과 반대".
- **판정**: **정당한 컷.** v4 의 w_eff 축소는 물리적으로 **거꾸로**였다(실측 두-상 봉우리는 평형 델타에서 broadening 으로 넓어진 종인데 v4 는 좁아진다고 함). v1.0.10 이 이를 제거하고 "폭=현상학적 자유 피팅" 으로 대체한 것은 **물리적으로 옳은 방향의 정정**. 코드도 use_w_eff 경로를 제거(헤더 L7, `_width` L305)해 문건과 정합.
- **★단, 미완의 뒷정리**: v4 는 w_eff 를 지우면서도 **"그럼 두-상 폭은 무엇이 정하나"의 코드 구현을 남기지 않았다**. 즉 틀린 축소식은 없앴으나 올바른 대체(broadening 반영 자유 폭 피팅)가 코드에 **구현 안 됨** → 코드는 결국 nRT/F 로 되돌아감(CRIT-2). 문건상 컷은 정당하나 **코드 귀결이 문건 의도(피팅 폭)를 못 따라감**이 남은 결함.

### v3 (Ch2 v3, 266줄 lean draft) → v1.0.10 = 실질 누락 없음(정당한 증보)
- v3 는 Bernardi 수지·∂U_j/∂T=ΔS/F·MSMR 추정경로·활성화 vs 반응 엔트로피 혼동경고의 **골격 초안**. v1.0.10 은 이 모두를 포함하고 통계열역학 본체(분배함수→점유분포→config/vib/elec)로 **확장**했다. v3 고유 물리 중 v1.0.10 에 빠진 것 = **발견 못 함**(v3 의 모든 핵심 식·경고가 v1.0.10 에 계승·심화). v3→v1.0.10 은 순증(정당).
- v3 의 부호 규약이 `ΔS=+nF·∂U/∂T`(n 명시), v1.0.10 은 `+F·∂U/∂T`(n=1 기준) — 규약 통일이지 누락 아님.

**cross-version 종합**: v4 의 유일한 물리 전개 차이(w_eff 축소 파생C)는 **틀린 물리라 제거가 정당**. v3 는 누락 없음. 따라서 cross-version 관점 결함은 "삭제된 물리의 부재"가 아니라 **"올바른 대체(broadening 폭 피팅)의 코드 미구현"** — 이는 CRIT-2 와 같은 문제로 수렴한다.

## 4. 5줄 요약

1. **초점**: code↔물리 정합 + 코드 직접 실행·그래프 개형 실측(FWHM·피크수·면적·부호)·Ch2 v3/v4 회귀.
2. **문제 수**: CRIT 2 · HIGH 1 · MED 3 · LOW 1 = 총 7건 (+ 무결 PASS 6영역 근거 기록).
3. **최중대**: CRIT-2 폭 w=nRT/F 고정 → CRIT-1 4-staging 단일 bell 병합. 문건은 "현상학적 자유 피팅 폭"이라 옳게 서술하나 코드가 미이행(문건-코드 gap).
4. **그래프 개형 판정**: 흑연 dQ/dV = **병합 단일 skewed bell(피크 1개)**, 4 staging spike 구분 안 됨(실측·커밋 그림 3종 일치). Omega 무관 FWHM=90mV = 순수 logistic 종 = 1차 두-상 near-delta 기대와 배치. 면적·부호·q_rev·seam·전자엔트로피는 PASS.
5. **오적발 가능성 자기표시**: MED-4(V vs x 축)는 두 경로 수치 동치 가능성 있어 오적발 위험 중간(절차 미이행은 확정, 결과 상이는 미확정). MED-5·MED-6·LOW-7 은 실측 확정이나 scope/게이트(γ=0) 고려 시 경중 재조정 여지. CRIT-1·2·HIGH-3 는 코드 실행·수치로 확정(오적발 위험 낮음). 초기 FWHM "이론 불일치" 의심은 내 상수(1.317) 오산으로 판명·자체 정정 → CRIT 아님(코드 종은 정확한 logistic 도함수).
