## 1. 결함 — 심각도 순

### [HIGH] 가이드 자체 모순: χ 의 tier/Phase 배정이 가이드 자신의 S-사슬 정의 및 Ch1 구조와 불일치

- **가이드 §1 tier 표**: χ 는 tier-2("히스·비대칭")로 분류, Phase B("히스·비대칭, 충·방전 pair") 에서 Ω·γ·Rn 과 함께 "개방"되며 주석은 "(= S2+S5)".
- **그러나 가이드 §3 S-사슬 표 자신**: S3("등온 다전위 꼬리, ln L_q vs V_drive 기울기 −χsF/RT")가 닫는 파라미터는 바로 **χ**(+부산물 r_a)이며, S2 는 R_n 만, S5 는 γ_j·Ω_j 만 닫는다 — **S2+S5 어디에도 χ 가 없다.** S3 는 오히려 Phase C("동역학 꼬리, rate-series")에 대응 배정되어 있다(가이드 §2 Phase C 주석 "(= S3)").
- 즉 가이드는 (i) tier 표에서 χ 를 Ω·γ·Rn 과 같은 "충·방전 pair" 데이터로 여는 그룹에 넣고, (ii) 같은 문서의 S-사슬 표에서는 χ 를 등온 다전위(=rate/전위 스윕, 유한전류 필요) 데이터로만 닫히는 것으로 정의해 놓았다 — 자기모순.
- **Ch1 원본과의 교차 확인**: Ch1 은 χ 를 처음부터 §sec:hys(히스테리시스, N3)가 아니라 §sec:lag(동역학 지연 길이, N7)에서만 정의·사용한다(χ_d=χ_split(χ,σ_d), eq:chid; ΔH_a^eff=ΔH_a−χ_d Ω, eq:dHeff). χ 는 Ω·γ·h_η 와 달리 sec:hys 어디에도 등장하지 않는다. 즉 Ch1 의 구조 자체가 χ 를 "동역학 꼬리" 파라미터로 분류하고 있어, 가이드의 tier-2/Phase-B 배정은 Ch1 의 절 구조와도 어긋난다.
- **실무 영향(G-usable)**: Phase B 단계에서 사용자가 "충·방전 gap" 데이터만으로 χ 를 열고 피팅을 시도하면, χ 의 실제 식별 신호(등온 다전위 꼬리 기울기, 유한전류 필요)가 그 데이터에 없어 χ 가 사실상 임의값에 머물거나 Ω·γ 와 공선형이 될 위험이 있다 — Phase 단계 분리(과식별 회피)라는 §2 의 설계 취지 자체를 χ 항목에서 무너뜨린다.
- **정정 제안**: (a) tier 표에서 χ 를 tier-2 대신 tier-3 인접(또는 "tier 2.5")으로 재분류하고 필요 데이터를 "등온 다전위 꼬리(rate-series)"로 정정, (b) Phase B 주석에서 χ 를 제외하고 "(= S2+S5, γ·Ω·Rn 만)"로 좁히거나, (c) Phase C 주석을 "(= S3+S4)"로 확장해 χ 를 Phase C 로 명시 이동. 세 방법 중 하나로 tier 표·Phase 서술·S-사슬 표 3자가 일치하도록 통일.

### [MED] Phase D "(= S4)" 주석과 S4 자신의 정의 불일치 가능성

- 가이드 §3 S-사슬 표: S4("다온도 꼬리, Arrhenius y(T) vs 1/T")가 닫는 파라미터는 **ΔH_a^eff, 절편**(동역학 활성화 엔탈피) — tier-3 항목이다.
- 가이드 §2 Phase D: "다온도·LCO 전자항, tier-4(dS_rxn·dS_a) + LCO 전자항 개방 ... (= S4)".
- tier-3(ΔH_a^eff)를 닫는 S4 가 tier-4(dS_rxn·dS_a, 평형 엔트로피)를 여는 Phase D 에 그대로 매핑되어 있어, [HIGH] 항목과 같은 종류의 tier↔S-사슬 어긋남이 반복된다. Phase C 의 "(=S3)"(실제로는 χ 를 닫음, dH_a 아님)까지 합쳐 보면 Phase{B,C,D} ↔ S{2+5, 3, 4} 매핑 전체가 한 칸씩 밀린 인상이다. [HIGH] 항목과 원인이 같으므로 같은 정정 라운드에서 함께 재정렬 권고(별도 이슈로 중복 처리하지 않음).

### [MED] ΔH_a 가시적 꼬리 문턱 수치: 가이드 "73–82 kJ/mol" vs Ch1 "≳7–8×10⁴ J/mol"

- 가이드 §2 Phase C: "★수치 지형 주의: ... 꼬리가 격자 문턱을 넘는 dH_a ≈ 73–82 kJ/mol ... (Ch1 tab:staging 하단 실현 크기 고지 참조)".
- Ch1 tab:staging 하단(L1998-2003, 가이드가 명시 지목한 바로 그 위치): "가시적 꼬리는 ΔH_a≳7--8×10⁴ J/mol" = 약 70–80 kJ/mol.
- 가이드 값(73–82)의 상한(82)이 Ch1 이 스스로 적어 둔 상한(≈80, "8×10⁴")을 2 kJ/mol 초과하고, 하한(73)도 Ch1 의 하한(70)보다 3 kJ/mol 높다. Ch1 은 "≳"(근사·이상) 표기라 거칠지만, 가이드가 "정확히 이 자리를 참조"라고 명시한 만큼 두 수치가 같은 계산에서 나왔다면 상한이 Ch1 표기를 벗어나는 것은 불일치다.
- 실무 영향은 작음(두 범위 모두 "표 초기값 40–48 은 문턱보다 한참 아래"라는 결론엔 영향 없음)이나, 피팅 시 dH_a 시작값을 "문턱 바로 위"로 놓으려는 사용자에게는 2–3 kJ/mol 차이가 실질적이다.
- **정정 제안**: 가이드의 73–82 산출 근거(코드 재계산 값인지, Ch1 문구를 반올림 없이 재인용한 것인지)를 밝히고, Ch1 본문의 "7–8×10⁴"도 동일 정밀도로 갱신하거나 가이드 쪽을 Ch1 표기(≈70–80)에 맞춰 통일.

### [LOW] 가이드 §3 S-사슬 표·§5 잔차표에 등장하는 "r_a" 미정의

- S3 행("χ (+부산물 r_a)")과 잔차 진단표("(1−r_a) 회계 누락")에 "r_a" 가 두 차례 등장하나, Ch1·Ch2·부록 어디에도 "r_a" 라는 기호의 정의(무엇의 비율인지, 어느 식에서 나오는지)가 없다. Ch1 의 표기 관례상 r_j 는 지연 변수(ξ_eq−ξ, eq:Lq 부근)로 이미 쓰이고 있어 "r_a" 와 혼동 소지도 있다.
- 실질적 물리 오류는 아니나(문서 간 모순이 아니라 가이드 내부의 정의 누락), G-usable 관점에서 사용자가 이 기호의 의미를 추적할 길이 없다.
- **정정 제안**: r_a 를 처음 쓰는 자리(S3 행)에 1줄 정의(예: "충방전 비가역 용량비" 등, 실제 의도에 맞게) 추가.
