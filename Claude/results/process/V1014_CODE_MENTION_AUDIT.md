# V1.0.14 P1.1 Step 3 — 코드-언급 전수 감사 (graphite_ica_ch1/ch2_v1.0.14.tex)

역할: 코드-언급 전수 감사자(작업 sub). 대상 파일은 감사만 하며 **수정하지 않았다**. 아래는 두 파일 전문 정독(ch1: L1–2934, ch2: L1–776) 기반 수집 결과다.

방침 재확인: 문건-코드 연계는 **코드가 문건의 식 번호를 참조하는 단방향**이어야 한다. 문건이 코드 변수명·함수명·클래스명·dict 키·소프트웨어 용어(생성자·서브클래스·메서드·하드코딩·폴백·facade 등)를 본문 프로즈에서 가리키는 것은 방침 위반 — 제거/물리 서술화하거나 Appendix "구현 대응표"로 이동해야 한다.

## 방법론 메모 (커버리지 선언 전에 먼저 명시)

- ch1(2934줄)·ch2(776줄) 전문을 offset/limit 분할 Read로 head→tail 빠짐없이 정독했다(ch1: 1–400,400–800,800–1200,1200–1600,1600–2000,2000–2400,2400–2700,2700–2934 8구간; ch2: 1–400,400–776 2구간).
- 보조로 `\code{`(ch1) / `\texttt{`(ch2) 정규식 grep을 라인 단위로 병행해 교차검증했다. grep raw count: ch1 `\code{` **155**(라인-매치 기준), ch2 `\texttt{` **6**. 본 표는 **한 문장/한 서식-대응 단위**로 인접 토큰을 묶어 행을 구성했으므로(예: `\code{func\_L\_q(...)}` 뒤에 이어지는 `\code{dH\_a\_use}` 등 5–7개 토큰을 한 처분 단위 행으로 묶음), 표의 "포함 토큰 수"를 각주로 병기해 실제 개별 `\code{`/`\texttt{` 호출 수(추정 총 ~205건, 표 항목 다수가 한 줄에 2–4개 토큰을 가져 라인-매치 155건보다 커짐)와 대조 가능하게 했다. **줄 단위 커버리지는 grep 라인 리스트 전건이 아래 표 어딘가에 귀속됨을 직접 대조로 확인했다** — 누락 0.
- `\code{}` 매크로는 ch1 preamble L99 `\newcommand{\code}[1]{\texttt{#1}}`에서 정의(ch2엔 이 매크로가 없어 ch2는 `\texttt{}`를 직접 사용).
